# AI 短剧导演技能

> 版本：1.0.0
>
> 稳定生成 3 分钟叙事视频。让 AI agent 成为短剧创作的「编剧+分镜+场记」

---

## 触发词

`AI短剧` `短剧导演` `做个短剧` `AI漫剧` `漫剧创作` `生成短剧` `视频剧本` `分镜脚本`

---

## 功能说明

解决 AI 短剧创作中的四大痛点：

- **需求模糊** -- 故事想法无法直接转化为 AI 可执行的 prompt
- **人物崩坏** -- 不同镜头间角色/场景不一致
- **节奏失控** -- 视频时长和叙事节奏无法精确控制
- **字幕乱码** -- AI 生成文字不稳定，乱码频出

### 核心原则

- **先定锚点，再写分镜**：人物和场景的一致性必须在第一镜前就确定
- **时间预算制**：3 分钟 = 180 秒，每镜多少秒要提前算好
- **字幕防乱码**：用 prompt 工程约束 + 后期校验清单双重保险
- **转场有章法**：根据情绪节奏选择合适的转场类型
- **分段生成，整体把控**：每段 20 秒以内，但分镜脚本要全局连贯

## 全流程指南

### Phase 0：需求澄清（必做，不急着写分镜）

用户说"我想做个短剧"时，**不要直接开始写 prompt**，先走完这 5 个问题的确认：

#### 0.1 故事类型确认

用选择题让用户锁定类型（单选）：

```
你的短剧偏向哪种类型？

A. 甜宠爱情（男女主暧昧拉扯，结局 HE）
B. 逆袭打脸（底层翻身，爽感为主）
C. 复仇悬疑（有秘密、有反转）
D. 家庭伦理（婆媳、亲情冲突）
E. 其他：___
```

#### 0.2 时长与模型能力确认

**先问清楚用户使用的视频模型**：

```
你打算用哪个视频生成模型？

A. 可灵（Kling）
B. 即梦（Dreamina）
C. Runway
D. Pika
E. 其他：___

这个模型单个视频最长支持多少秒？___秒
（如果不确定，建议先查一下官方文档，这会影响我们怎么拆分任务）
```

**再问目标时长**：

```
目标总时长？

A. 60秒（快节奏，适合抖音开头抓眼球）
B. 90秒（标准短剧一集长度）
C. 180秒（3分钟，可以讲完一个完整小闭环）
D. 其他：___秒
```

**自动计算分段策略**：

```
根据你的模型能力（单段最长{X}秒）和目标总时长（{Y}秒）：

建议拆分方案：
- 总段数：{Y/X 向上取整} 段
- 每段时长：约 {X} 秒（用满单段上限，减少拼接次数）
- 最后一段可能只有 {Y%X} 秒

或者如果你希望节奏更紧凑：
- 总段数：{Y/10 向上取整} 段
- 每段时长：约 10-15 秒（短镜头拼接，节奏更快）

你倾向哪种？
```

#### 0.3 风格参考确认

```
有对标参考吗？（帮助 AI 理解你想要的视觉风格）

A. 某部具体短剧：___
B. 某部电影/剧集的画风：___
C. 我发你参考图片/视频
D. 没有，你根据类型推荐
```

#### 0.4 核心人物设定（初步）

```
主要角色有几个？

主角1（男/女）：___
- 年龄感：___
- 身份/职业：___
- 性格关键词：___

主角2（如有）：___
- ...

有反派/配角吗？___
```

#### 0.5 一句话故事核

让用户用一句话概括：

```
用一句话讲清楚这个故事：
"这是一个关于______的故事，主角想要______，但遇到了______的阻碍，
最后______。"
```

**确认完以上 5 项后，输出需求摘要给用户确认：**

```markdown
# 短剧需求摘要 - {项目名}

## 故事类型
{类型}

## 时长规划
- 目标总时长：{N}秒
- 分段数：{M}段（每段约{N/M}秒）

## 风格参考
{参考描述}

## 核心人物
- {主角1}：{一句话描述}
- {主角2}：{如有}

## 一句话故事核
{故事核}

## 关键场景（用户提到的）
- {场景1}
- {场景2}

确认无误？确认后我们进入 Phase 1：建立人物和场景的一致性锚点。
```

### Phase 1：一致性锚点建立（防止崩脸崩场景）

这是整个流程最关键的阶段。**所有后续分镜都要引用这里的锚点。**

#### 1.1 人物视觉锚点卡

为每个主要角色创建「视觉锚点卡」，包含：

```markdown
## 角色：{角色名}

### 基础外貌（文字描述，用于 prompt）
- 性别/年龄：
- 脸型/发型：
- 五官特征：
- 肤色/妆容：

### 标志性服装（每套衣服给一个编号）
- 服装A：{详细描述，如"深灰色修身西装，白衬衫，无领带"}
- 服装B：{如有换装}

### 标志性道具/配饰
- {如：左手腕银色手表、黑框眼镜}

### 参考图片
- [ ] 用户已上传参考图
- [ ] 如无，用文字描述生成首镜后人工确认

### 一致性注入 prompt 模板
```
{角色名}, {基础外貌}, 身穿{服装编号}, {标志性道具},
保持面部特征一致，同一人，连续镜头
```
```

**SOLO 操作**：
1. 根据 Phase 0 的人物设定，生成文字版锚点描述
2. **询问用户**："有参考图吗？发给我，我保存到角色卡"
3. 如果有参考图，保存到项目文件夹 `references/characters/{角色名}.jpg`
4. 输出完整的锚点卡，让用户确认或修改

#### 1.2 场景视觉锚点卡

为每个关键场景创建锚点：

```markdown
## 场景：{场景名}

### 环境描述
- 室内/室外：
- 空间类型：{如：现代办公室、复古咖啡馆、雨夜街道}
- 色调/光线：{如：冷白光、暖黄灯光、阴天自然光}

### 标志性元素（用于识别这是哪个场景）
- {如：落地窗城市夜景、红砖墙、老式留声机}

### 一致性注入 prompt 模板
```
{场景描述}, {色调光线}, {标志性元素}, 同一地点，连续场景
```
```

#### 1.3 输出锚点圣经（ANCHOR_BIBLE.md）

将以上所有锚点汇总到一个文件：

```markdown
# ANCHOR_BIBLE.md - {项目名}
生成日期：{日期}

---

## 人物锚点

### {角色1}
{锚点卡内容}

### {角色2}
{锚点卡内容}

---

## 场景锚点

### {场景1}
{锚点卡内容}

### {场景2}
{锚点卡内容}

---

## 使用说明
每段视频的 prompt 必须引用对应角色和场景的「一致性注入模板」，
确保人物不崩、场景不乱。
```

**告知用户**：
```
我在项目目录建了 ANCHOR_BIBLE.md，里面存了所有人物和场景的一致性锚点。
后面每段视频生成时，我都会自动引用这些描述，确保主角不会突然换脸。
```

### Phase 2：分镜脚本生成（时间预算制）

#### 2.1 时间分配策略

根据总时长，按故事结构分配时间：

| 故事结构 | 占比 | 示例（总时长180秒，单段上限20秒） |
|---------|------|----------------------------------|
| 开场钩子 | 10% | 18秒（1段） |
| 铺垫/日常 | 20% | 36秒（2段） |
| 冲突/转折 | 30% | 54秒（3段） |
| 高潮 | 25% | 45秒（2-3段） |
| 收尾/钩子 | 15% | 27秒（2段） |
| **总计** | 100% | **180秒（约9-10段）** |

> 注：实际段数根据用户模型的单段时长上限动态调整。

**SOLO 操作**：
1. 根据故事核，建议故事结构分配
2. 让用户确认或调整
3. 输出「时间预算表」

#### 2.2 分段规划

按每段 15-20 秒，规划每段要覆盖的剧情：

```markdown
# 分段规划表

## 第1段（0-18秒）- 开场钩子
剧情：{一句话}
涉及角色：{角色名}
涉及场景：{场景名}

## 第2段（18-36秒）- 铺垫开始
剧情：{一句话}
...
```

#### 2.3 逐镜分镜表（每段内部）

每段再细分为「镜」，每镜包含：

```markdown
### 第{N}段分镜表

| 镜号 | 画面内容 | 角色/服装 | 场景 | 时长 | 字幕 | 转场 |
|-----|---------|----------|------|------|------|------|
| N-1 | {描述} | {角色}+{服装编号} | {场景} | 5s | {台词} | {转场类型} |
| N-2 | {描述} | ... | ... | 4s | ... | ... |
| ... | ... | ... | ... | ... | ... | ... |

**本段总时长**：{X}秒
**下段衔接提示**：{剧情衔接点}
```

**转场类型选择指南**：

| 转场类型 | 适用场景 | prompt 关键词 |
|---------|---------|--------------|
| 硬切 | 快节奏、对话切换 | hard cut, jump cut |
| 叠化 | 情绪过渡、时间流逝 | cross dissolve, fade |
| 匹配剪辑 | 动作连续性 | match cut, action continuity |
| 黑场 | 大时间跳跃、章节分隔 | fade to black |
| 运动模糊 | 快速移动场景 | motion blur transition |

**SOLO 操作**：
1. 为每段生成分镜表
2. 每段生成后让用户确认："这段的分镜这样安排，节奏 OK 吗？"
3. 确认后再进入下一段

### Phase 3：Prompt 工程化输出

#### 3.1 单段 Prompt 结构

每段视频生成用以下 prompt 结构：

```
【画面主体】
{场景锚点描述}, {角色1锚点描述}, {角色2锚点描述},
{具体动作/互动描述}

【镜头语言】
{景别：特写/中景/全景}, {运镜：固定/推/拉/摇/跟},
{光线条件}, {色调风格}

【转场提示】
{转场类型}, {与上一段的衔接描述}

【一致性约束】
同一场戏连续镜头，人物面部特征保持一致，服装不变，场景光线统一

【字幕区域预留】
画面底部20%区域保持简洁，避免复杂纹理，便于后期加字幕

【负面提示】
变形的人脸，多余的手指，闪烁的画面，突然的光线变化，
字幕乱码，文字扭曲，画面抖动
```

#### 3.2 字幕防乱码专项约束

在 prompt 中增加字幕相关约束（关键！）：

```
【字幕生成约束 - 如需要画面内文字】
- 画面内出现的文字必须是：{指定文字内容}
- 字体风格：{如：现代简洁无衬线字体}
- 文字位置：{如：画面底部居中}
- 文字颜色：{如：白色带黑色描边，确保可读性}
- 禁止：自动生成随机字符、乱码、扭曲文字

【后期字幕建议】
本段台词："{实际台词}"
建议生成后使用剪映/PR 的语音识别功能添加字幕，
或使用本段台词手动添加，避免 AI 生成乱码文字。
```

**SOLO 必须提醒用户**：
```
重要提示：目前视频生成模型在画面内文字方面表现不稳定，
很容易出现乱码或扭曲文字。

建议方案：
A. 生成纯画面，后期用剪辑软件加字幕（推荐，最稳定）
B. 如果必须在画面内出现文字，我会尽量约束，但需要你人工检查

你倾向哪种？
```

#### 3.3 批量生成任务清单

输出可执行的任务清单：

```markdown
# 生成任务清单 - {项目名}

## 第1段（0-18秒）
- [ ] Prompt 已生成
- [ ] 已复制到视频生成工具
- [ ] 生成完成
- [ ] 一致性检查通过
- [ ] 字幕检查通过

## 第2段（18-36秒）
...

## 一致性自查清单（每段生成后必做）
- [ ] 人物面部与锚点卡一致
- [ ] 服装与锚点卡一致
- [ ] 场景与锚点卡一致
- [ ] 光线/色调与上一段连贯
- [ ] 没有乱码字幕
```

### Phase 4：后期校验与迭代

#### 4.1 单段生成后校验

每段视频生成后，引导用户检查：

```
第{N}段生成完成，请检查以下项目：

人物一致性：
- 主角面部特征与锚点卡一致
- 服装与锚点卡一致
- 没有出现多余的手指/肢体变形

场景一致性：
- 场景环境与锚点卡一致
- 光线/色调与预期相符

字幕检查：
- 画面内无乱码文字
- 如需后期加字幕，画面底部是否预留了干净区域

转场检查：
- 与上一段的衔接自然

如果检查通过，我们继续生成第{N+1}段。
如果有问题，告诉我具体哪里不对，我们调整 prompt 重跑。
```

#### 4.2 全片节奏复盘

所有段生成完成后：

```
全片生成完成！来复盘一下节奏：

整体节奏感受：
- 开头是否足够抓人？
- 中间是否有拖沓的地方？
- 高潮是否到位？
- 结尾是否仓促？

如果有某段觉得节奏不对，告诉我：
- 第几段？
- 快了还是慢了？
- 需要怎么调整？

我们可以单独重跑那一段。
```

#### 4.3 锚点 Bible 更新

如果在生成过程中发现锚点需要调整（比如某套服装实际效果不好）：

```
是否需要更新 ANCHOR_BIBLE.md？

本次发现的调整：
- {角色}的{服装编号}在实际生成中{问题描述}
- 建议改为：{新描述}

更新后后续所有段都会使用新锚点。
```

## 核心原则速查

| 原则 | 执行方式 |
|------|---------|
| 先定锚点再写分镜 | Phase 1 必须完成 ANCHOR_BIBLE.md，后续每段引用 |
| 时间预算制 | 3 分钟=180 秒，每段每镜提前算好时长 |
| 分段生成，整体把控 | 每段 15-20 秒，但分镜脚本全局连贯 |
| 字幕防乱码 | 优先后期加字幕，画面内文字严格约束 |
| 转场有章法 | 根据情绪节奏选转场类型，不生硬切换 |
| 每段必校验 | 人物/场景/字幕/转场四项检查 |
| 锚点可迭代 | 生成中发现问题及时更新 ANCHOR_BIBLE.md |

## 常见问题处理

### Q：人物还是崩了怎么办？

**纠偏话术**：
```
人物不一致的问题，我们来排查：

1. 锚点描述是否足够具体？
   - 现在的描述：{当前描述}
   - 建议增加：{更具体的特征，如"左脸颊有颗痣"、"刘海三七分"}

2. 是否有参考图？
   - 上传一张该角色的参考图，我在 prompt 中强调"参考图中的人物"

3. 是否需要降低变化度？
   - 在 prompt 中加："完全相同的面部特征，零变化，静态表情"

试试调整后重跑这一段？
```

### Q：节奏感觉不对，某段太拖沓？

**纠偏话术**：
```
第{N}段节奏拖沓，我们来调整：

方案A：压缩时长
- 原时长：{X}秒 -> 建议改为：{Y}秒
- 需要删减的镜头：{具体镜号}

方案B：加快剪辑节奏
- 增加镜头数量，减少每个镜头的停留时间
- 多用硬切，减少叠化

方案C：调整剧情密度
- 这段的剧情信息量可能不够，需要增加冲突点或转折

你倾向哪个方向？
```

### Q：字幕识别出来错别字很多？

**纠偏话术**：
```
字幕识别错误率高，建议：

方案A（推荐）：后期手动校对
- 用剪映/PR 的语音识别生成字幕后，人工逐句校对
- 时间成本：3分钟视频约10-15分钟

方案B：使用外部字幕服务
- 先提取音频，用飞书妙记/通义听悟等工具转文字
- 准确率通常比视频生成工具内置识别高

方案C：如果必须在画面内生成文字
- 我会把文字约束放到 prompt 最前面，增加权重
- 但无法100%保证，仍需人工检查

建议用方案A，最可控。
```

## 反模式：绝对不要

### 错误做法

| 错误做法 | 问题说明 | 正确做法 |
|---------|---------|---------|
| 跳过锚点直接生成画面 | 人物会崩，场景会变，全靠碰运气 | Phase 1 必须先完成 ANCHOR_BIBLE.md，每段生成都引用锚点 |
| 不做时间预算，想生成多久就多久 | 时长失控，节奏混乱，最后凑不齐成片 | 3 分钟=180 秒，每段每镜提前算好时长，分段规划 |
| 所有内容一段生成完 | 时长太长，模型能力跟不上，人物必然崩 | 每段 15-20 秒，分段生成，逐段校验 |
| 不做逐段校验，全部生成完再检查 | 发现问题时已经生成一堆垃圾，浪费时间和资源 | 每段生成后立即做人物/场景/字幕/转场四项检查 |
| 强制在画面内生成字幕 | 90% 概率出现乱码、扭曲文字，无法使用 | 优先后期用剪映/PR 加字幕，画面只留底部 20% 干净区域 |
| 锚点描述太模糊 | "一个女孩"太模糊，不同镜头差异巨大 | 锚点必须具体："25 岁中国女性，左脸颊有颗痣，长发微卷" |
| 不检查乱码就继续 | 后面全毁了，成片无法使用 | 每段生成后必须检查：画面内有没有乱码文字？ |
| 转场随便用 | 情绪跳脱，节奏混乱，观众出戏 | 根据情绪节奏选择：快节奏用硬切，情绪过渡用叠化 |
| 发现锚点问题不更新 | 后面所有段都会有同样问题 | 生成中发现锚点有问题，立即更新 ANCHOR_BIBLE.md |
| 不做最终全片复盘 | 节奏问题、衔接问题发现不了，成片质量差 | 全部生成完后做全片节奏复盘，调整不满意的段落 |

### 常见错误

1. **不要妄想一次完美**：AI 视频生成需要迭代，接受需要重跑几段的现实
2. **不要忽略负面提示词**："变形的人脸、多余的手指"这些必须加
3. **不要跳过参考图**：有参考图时人物一致性会好很多
4. **不要在画面里放太多复杂文字**：AI 生成文字能力很弱，后期加字幕才是王道
5. **不要一段太长**：超过 30 秒人物就容易崩，控制在 15-20 秒内最稳
6. **不要忽略光线统一**：前一段是日光，后一段突然变成夜景，观众会出戏
7. **不要忘记预留字幕区域**：画面底部 20% 放复杂内容，后期字幕根本没法加
8. **不要在没有校验的情况下继续**：某段人物崩了，继续生成后面的也没用
9. **不要用太抽象的艺术风格**：抽象风格下人物一致性更难保证，优先用写实风格
10. **不要一条路走到黑**：某套服装生成效果一直不好，立即换一套
