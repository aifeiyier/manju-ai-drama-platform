## 功能说明

剪辑合成模块是 AI 漫剧创作链路的第四步。接收素材生成模块的视觉和音频素材，执行音画对齐、转场效果添加、字幕生成和质量优化，输出可播放的成片。

核心能力：
- **音画对齐** -- 自动对齐视觉素材和音频素材的时间轴
- **自动转场** -- 基于情绪差异自动选择最合适的转场效果
- **SRT 字幕生成** -- 根据分镜表的台词自动生成标准 SRT 字幕文件
- **质量优化** -- 4 档质量预设，自动估算文件大小

## 10 种转场效果

| 转场效果 | 英文ID | 描述 |
|---------|--------|------|
| 淡入淡出 | fade | 画面逐渐消失/出现，最常用的基础转场 |
| 溶解过渡 | dissolve | 两个画面叠加融合过渡 |
| 向右擦除 | wipe_right | 新画面从左向右擦除旧画面 |
| 向左擦除 | wipe_left | 新画面从右向左擦除旧画面 |
| 向上滑动 | slide_up | 新画面从下方向上推入 |
| 向下滑动 | slide_down | 新画面从上方向下推入 |
| 放大切入 | zoom_in | 画面放大切入下一个镜头 |
| 缩小切入 | zoom_out | 画面缩小切入下一个镜头 |
| 故障效果 | glitch | 数字故障风格转场，适合悬疑/科幻 |
| 模糊过渡 | blur | 画面模糊后切换到下一个镜头 |

## 自动转场选择逻辑

### 基于情绪差异的转场选择

转场类型根据相邻两个镜头的情绪差异自动选择：

| 情绪差异范围 | 选择转场 | 说明 |
|------------|---------|------|
| > 0.7 | glitch（故障效果） | 情绪剧烈变化，需要强烈的视觉冲击 |
| > 0.4 | dissolve（溶解过渡） | 情绪有明显变化，需要柔和过渡 |
| 全景 -> 特写 | zoom_in（放大切入） | 镜头类型变化，用缩放衔接 |
| 特写 -> 全景 | zoom_out（缩小切入） | 镜头类型变化，用缩放衔接 |
| 其他情况 | fade（淡入淡出） | 默认转场 |

### 情绪强度值

| 情绪 | 强度值 |
|------|--------|
| 平静 | 0.1 |
| 释然 | 0.2 |
| 温馨 | 0.3 |
| 满足 | 0.3 |
| 好奇 | 0.4 |
| 期待 | 0.4 |
| 羞涩 | 0.5 |
| 感动 | 0.6 |
| 惊讶 | 0.7 |
| 疑惑 | 0.7 |
| 紧张 | 0.75 |
| 担忧 | 0.75 |
| 愤怒 | 0.85 |
| 恐惧 | 0.85 |
| 激昂 | 0.9 |
| 震惊 | 0.95 |

**情绪差异** = |前镜情绪强度 - 后镜情绪强度|

### 手动转场指定

除了自动选择外，支持以下方式指定转场：

| 方式 | 说明 |
|------|------|
| `auto`（默认） | 基于情绪差异自动选择 |
| 字符串 | 所有转场使用同一种效果 |
| 数组 | 按数组顺序循环使用指定的转场效果 |

## 4 档质量预设

| 预设 | 视频码率 | 音频码率 | 帧率 | 分辨率 | 适用场景 |
|------|---------|---------|------|--------|---------|
| low | 2Mbps | 128kbps | 24fps | 720p | 预览、快速测试 |
| medium | 5Mbps | 192kbps | 30fps | 1080p | 标准发布 |
| high | 10Mbps | 320kbps | 30fps | 1080p | 高质量发布（推荐） |
| ultra | 16Mbps | 320kbps | 60fps | 4K | 最高画质 |

### 质量优化处理

所有预设都会应用以下优化：

- 颜色校正
- 锐化处理
- 降噪处理
- 帧率稳定化

### 文件大小估算

```
文件大小(MB) = (视频码率Mbps + 音频码率Mbps) * 时长(秒) / 8
```

| 预设 | 60秒估算 | 90秒估算 | 180秒估算 |
|------|---------|---------|----------|
| low | ~16MB | ~24MB | ~48MB |
| medium | ~39MB | ~58MB | ~116MB |
| high | ~77MB | ~116MB | ~231MB |
| ultra | ~123MB | ~184MB | ~368MB |

## 音画对齐

### 对齐逻辑

1. 遍历所有视觉素材
2. 为每个视觉素材查找对应的配音（通过 shot_id 匹配）
3. 计算音画偏移量
4. 偏移量 <= 40ms 标记为 `ok`，> 40ms 标记为 `warning`
5. 如果 warning 超过 3 个，整体标记为 `fail`

### 对齐报告格式

```json
{
  "audio_video_sync": "pass",
  "sync_offsets": [
    { "shot_id": 1, "offset": 0.01, "status": "ok" }
  ],
  "issues": []
}
```

## SRT 字幕生成

### 生成规则

1. 遍历分镜表中所有有台词（dialogue）的镜头
2. 根据镜头时长计算开始时间和结束时间
3. 生成标准 SRT 格式字幕

### SRT 格式

```
1
00:00:00,000 --> 00:00:05,000
主角: 这是什么地方？

2
00:00:05,000 --> 00:00:10,000
主角: 好强的力量！
```

### 时间格式

`HH:MM:SS,mmm`（时:分:秒,毫秒）

### 字幕输出

- **文件路径**：`output/subtitles.srt`
- **格式**：SRT
- **包含角色名**：是（格式为 `角色名: 台词`）

---

## 时间线生成

### 时间线结构

为每个镜头生成时间线条目：

```json
{
  "shot_id": 1,
  "start_time": 0,
  "duration": 5,
  "end_time": 5,
  "visual": "assets/visual/shot_1.png",
  "audio": "assets/audio/voiceover_1.mp3",
  "bgm": "assets/audio/bgm.mp3",
  "transition": null,
  "camera_movement": "固定",
  "shot_type": "全景"
}
```

- `bgm` 仅在第一个镜头时设置（后续镜头继承）
- `transition` 为前一个转场效果（第一个镜头为 null）

```json
      "video_quality": "excellent",
      "audio_quality": "excellent",

      "sync_details": { ... },
      "quality_settings": { ... },
      "issues": []
    }
  }

```

### 整体评分计算

| 因素 | 评分影响 |
|------|---------|
| 基础分 | 0.85 |
| 音画同步失败 | -0.2 |
| 音画同步有 warning | -0.05 |
| 视频质量 excellent | +0.05 |
| 视频质量 fair | -0.05 |
| 视频质量 poor | -0.1 |

最终评分限制在 0-1 范围内。

## 使用方法

### SOLO 执行步骤

1. 接收素材生成模块的视觉素材和音频素材
2. 接收故事策划模块的分镜表
3. 执行音画对齐，生成对齐报告
4. 应用转场效果（默认 auto 模式）
5. 生成 SRT 字幕（默认开启）
6. 执行质量优化（默认 high 预设）
7. 生成时间线
8. 计算整体评分
9. 输出完整的剪辑报告

### 可用操作

| 操作 | 说明 |
|------|------|
| compile | 执行完整的剪辑合成流程 |
| list_transitions | 列出所有可用的转场效果及描述 |
| list_quality_presets | 列出所有质量预设及详细参数 |
| preview_edit | 预览指定镜头的剪辑效果 |

## 反模式

| 错误做法 | 问题说明 | 正确做法 |
|---------|---------|---------|
| 不做音画对齐直接合成 | 台词和画面不同步 | 必须执行音画对齐，检查偏移量 |
| 所有转场都用同一种 | 节奏单调，缺乏变化 | 使用 auto 模式，根据情绪差异选择转场 |
| 不生成字幕 | 观众无法理解台词 | 默认开启自动字幕生成 |
| 使用 low 质量发布 | 画质模糊，影响观感 | 发布时至少使用 medium，推荐 high |
| 忽略质量报告的 warning | 小问题累积成大问题 | 检查所有 warning，必要时调整 |
| 转场时长不统一 | 节奏不协调 | 所有转场默认 0.5 秒时长 |
